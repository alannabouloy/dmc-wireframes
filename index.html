User Flow

  1. Landing Page

    On the landing page, the user will select either the 'new user' button, or the button for returning users.

    • If they select the 'new user' option they will be taken to the registration page.
    • Otherwise they will go straight to a login page.

  2. Login Page

    The user will be asked to fill out the login form with a username and password:

    • If both are valid, then they will be redirected to their personal dashboard.
    • If the password is invalid, they will get an error message saying that they entered the wrong password.
    • If the username is invalid, they will get an error message saying that they entered the wrong username.

  3. Registration Page

    If the user says they are new, they will be taken to a registration page where they will fill out a form with a username, password, and email.

    • If the username, password, and email are all valid, they will be added as a user and taken to their new dashboard.
    • If the username is too short, or nonexistent, they will get an error message telling them that they must have a username of 5+ characters
    • If the username is taken by another user, they will get an error message telling them that the username has already been selected.
    • If the password is not valid, they will get an error message telling them that they must have a password of 8+ characters
    • If email is not a valid email, they will get an error message saying that they must give a valid email account
    • If the email corresponds to another account on file, they will get a message notifying them that the email is already taken and asking if they would like to login

  4. Dashboard

    Upon logging in, users will be taken to a main dashboard where they will be able to view all active campaigns for their account and have the option to add or join a campaign.

    • If they are a new user, this dashboard will be empty except for the buttons to add or join a campaign
    • If they are a returning user, they will see all active campaigns associated with their account.
    • If the user clicks on a campaign tab they will be redirected to that campaign's dashboard.
    • If the user clicks on the 'Add Campaign' button, they will be redirected to a form page to add a new campaign
    • If the user clicks on the 'Join Campaign' button, they will be taken to a page of all public campaigns available to join

  5. Campaign Dashboard

    Upon clicking on an active campaign tab, users will be redirected to that campaign's dashboard where they will see tabs for all notes associated with that campaign, any NPCs added to the campaign, and the current campaign bank account. There will be an Add Note button and an Add NPC button at the bottom of the page.

    • Users will be able to click on a note tab and be redirected to a detailed Note Page.
    • Users will be able to click on an NPC tab and be redirected to a detailed NPC page.
    • Users will be able to view how much money the party has currently, and make withdrawals and deposits to the bank which update the amount.
    • Users will be able to click on an Add Note button that takes them to a form page to add a new note.
    • Users will be able to click on an Add NPC button that takes them to a form page to add a new NPC.
    • If there are no notes in the campaign, User will be able to see the Add Note and Add NPC buttons on their dash.
    • User will be able to select tags to filter notes by.
